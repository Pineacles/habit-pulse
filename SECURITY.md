# Security Policy

## Supported Versions

Only the latest release on the `main` branch is actively maintained.

## Reporting a Vulnerability

Please **do not** file public GitHub issues for security vulnerabilities.

To report a security issue:

1. **GitHub**: Use the [private vulnerability reporting](../../security/advisories/new) feature on this repository (Settings → Security → Report a vulnerability).
2. **Email**: Send details to the maintainer at the email address listed on the GitHub profile.

Include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You will receive an acknowledgement within 72 hours and a follow-up once the issue has been assessed.
